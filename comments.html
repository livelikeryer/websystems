HTML Element Description and Purpose
<!DOCTYPE html> Specifies the document type (HTML5) for the page. It tells the browser which version of HTML the page is using.
<html> The root element that wraps the entire HTML content. It represents the HTML document.
<head> Contains metadata and links to external resources. It doesn't display any visible content on the web page.
<meta> Provides metadata information about the document, including character encoding and viewport settings. It's used for SEO and compatibility.
<link> Links to external resources, such as stylesheets (CSS), to apply styles to the web page. It's often used to connect CSS files.
<title> Sets the title of the web page, which is displayed in the browser's title bar or tab. It helps identify the page and improves SEO.
<body> Contains the visible content of the web page, including text, images, links, and other elements. It represents the main content area.
<header> Represents the header section of the web page, typically containing the website's name or logo and navigation links.
<nav> Contains navigation links for the website, such as a menu. It helps users navigate the site's different sections.
<ul> Represents an unordered list, often used to create a bulleted list. It can be used for organizing content, such as navigation menus.
<li> Represents a list item within an ordered or unordered list. It's used to define individual items within a list.
<a> Anchor links used for navigation within the website or linking to external web pages. It allows users to click and navigate to different pages or sections.
<main> Contains the main content of each web page, including articles, sections, or other content. It represents the primary content area.
<section> Represents a section of content within the main content area. It's used to group related content and provide semantic structure.
<h2> Subheading used to denote section titles or headings within a page. It helps structure the content and improve readability.
<p> Paragraphs of text used to provide content and information. It's the standard way to include textual content on a web page.
<footer> Represents the footer section of each web page, often containing copyright information and other footer content.


Overall, the CSS selectors and rules are used to style different elements of the web pages to create a visually appealing and organized layout. Styles are applied to elements like body, header, nav, a, and more to ensure a consistent and cohesive design throughout the website. CSS resets default styles and defines typography, spacing, and colors to enhance readability and aesthetics.

Aesthetics -

The design aims for a professional, clean, and accessible look while maintaining a visually pleasing layout. The color contrast and clear typography choices enhance readability, and the layout ensures that users can easily navigate and consume the content. Overall, it's a balanced design that focuses on both aesthetics and functionality. I wanted a more simple look to the website as I feel like it is visually pleasing and something that isn't too flashy for peoples likings.


I used a combination of contrasting colors to create a visually appealing design. The header and footer have a dark background (#333) to provide contrast with white text for better readability. The body content area has a light background to make the content stand out.


The header section is designed to be bold and eye-catching. It features a dark background color, white text, and centered alignment. The website title and navigation menu are prominent and easy to find.


Paragraphs are styled for improved text readability, with a comfortable line height and spacing. Margins and padding are used to create appropriate spacing around text content.




Accessibility -

  • Semantic HTML elements, such as header, nav, main, and footer, have been used appropriately to provide a meaningful structure to the content. This helps screen readers and other assistive technologies interpret the page's layout and content correctly.

  • Used a simple colour scheme so people with color blindness don't feel like they are being left out.

  • Users won't have any trouble reading the text because the font size has been kept at an adequate size for the screen. Text throughout paragraphs is readable. For the best readability, I left the font at its default setting.

  • For the reader to be able to read the content without getting confused, I used appropriate line and letter spacing.
